## Overview

The OAGI API Gateway provides endpoints for interacting with OAGI workers and managing file uploads for screenshot-based task automation.

**Base URL:** `https://api.agiopen.org`

## Available Endpoints

<CardGroup cols={2}>
  <Card title="Generate" icon="robot" href="/api-reference/endpoint/generate">
    Generate responses using OAGI workers (oagi_first, oagi_follow, oagi_task_summary)
  </Card>
  <Card title="File Upload" icon="upload" href="/api-reference/endpoint/file-upload">
    Get presigned URLs for uploading screenshots
  </Card>
  <Card title="Message" icon="message" href="/api-reference/endpoint/message">
    Analyze chat messages with screenshots to get possible actions
  </Card>
</CardGroup>

## Authentication

All API endpoints are authenticated using an API key passed in the `x-api-key` header.

```bash
curl -X POST https://api.agiopen.org/v1/generate \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ ... }'
```

You can obtain an API key from the [OpenAGI Developer Dashboard](https://developer.agiopen.org).

## Error Handling

All endpoints return consistent error responses:

```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable error message"
  }
}
```

### Common Error Codes

| Status | Description |
|--------|-------------|
| 400 | Bad Request - Invalid request parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 422 | Validation Error - Request body validation failed |
| 429 | Rate Limit Exceeded - Too many requests |
| 500 | Internal Server Error |
| 503 | Service Unavailable |
